Development Team Project: Case Study Summary

IoT Based Smart Security and Home Automation System

Thursday, 31 October 2024, 10:35 PM

Introduction

In this post, we explore several case studies and carefully analyse each one to select a specific focus for our ADTool group report. This post discusses an article by Kodali et al. (2016) titled "IoT-Based Smart Security and Home Automation System." To read the full group report, click on the following link: Development Team Project: Design Document—Group Submission.}

The article discusses Internet of Things (IoT) applications in smart security systems and home automation. It describes a system that enables remote monitoring and control. The focus is on a smart wireless home security system designed to alert users in case of unauthorised entry or trespassing. This feature enables users to respond quickly and raise an alarm.

The triggering ability also allows the user to control various appliances within the house. This can also enable users to decide when they return home and have the system activate appliances such as the TV.

This system is relatively cost-effective, compatible with any platform, and can be accessed from anywhere. It utilises PIR motion sensors to detect individuals' presence and can be controlled via a mobile phone keypad. The IoT security system also provides instructions on how to set it up, lists the necessary components, and suggests potential future improvements. The system has the following key features and benefits.

  • Trespass detection – detect unauthorised entries
  • Alert mechanism – send alerts to the user via wifi connectivity
  • Optional alarm – the user can trigger the alarm
  • Enhanced security – providing real-time monitoring and alerts via phone call
  • Remote access - the user receives alerts allowing control of the system remotely
  • Automation – the integration of security features

(Kodali et al., 2016)

Notes:

“Since the launchpad sends a voice call to only a particular number which is present in the web API there is no need to worry about security leakage as the system cannot be accessed by any other unauthenticated user. This in turn increases the fidelity of the security system.”

The potential vulnerability here is that voice calls can be hacked, or if the system has been breached, threat actors can override it.

“The use of wifi enabled launchpad in the system enables the system to be controlled from any part of the globe contrary to Blue-tooth controlled or IR remote controlled existing home automation solutions that too without any net connectivity in the phone.”

This has the potential threat actors to attack the system from any jurisdiction.

Possible vulnerabilities:

  • Weak authentication

    Numerous IoT devices possess weak, hard-coded, or easily guessable default passwords that users frequently neglect to alter. This vulnerability allows attackers to quickly obtain unauthorised access (Mirza, 2024; Trendmicro, 2020).

  • Insecure Network Service

    IoT devices often use unsecured protocols, such as HTTP, FTP, and Telnet, to send data unencrypted, which makes the data susceptible to interception (Mirza, 2024).

  • Outdated software or firmware

    Numerous IoT devices operate on obsolete operating systems or firmware that manufacturers no longer support, making them vulnerable to known exploits (Mirza, 2024; Bodnar, 2023).

  • Botnet

    Compromised IoT devices can be exploited to create larger botnets that launch distributed denial-of-service (DDoS) attacks, as exemplified by the notorious Mirai botnet attack (Balbix, 2020; Trendmicro, 2020).

  • Data breaches

    Attackers may exploit vulnerabilities to access sensitive data on or transmitted by IoT devices (Mirza, 2024; Bodnar, 2023).

  • User awareness

    Many users do not have security awareness, leaving smart devices vulnerable to attacks (Trendmicro, 2020).

  • Lack / insufficient updates

    Numerous IoT devices do not have adequate update processes, or users often neglect to apply updates, which exposes them to emerging threats (Bodnar, 2023).

Addressing these risks requires a holistic strategy that encompasses collaboration among manufacturers, end-users, and information technology specialists. This collaboration is essential to ensure that adequate security protocols are established and maintained within the Internet of Things (IoT) ecosystem.
