Collaborative Learning Discussion - Unit 3

Monday, 13 May 2024, 6:41 PM

In my initial post, I presented a concise introduction to the Open Web Application Security Project (OWASP), a non-profit organisation that promotes awareness of significant web application vulnerabilities. The organisation brings together a diverse community of contributors who collaborate to improve web security by identifying and exposing weaknesses and vulnerabilities (OWASP, 2020).

This post focuses on one of the top 10 application risks identified by the Open Web Application Security Project (OWASP). This risk is particularly noteworthy as it falls under a new category introduced in 2021, namely A04:2021-Insecure Design. This category highlights the dangers of suboptimal development practices, leading to design flaws within the application system (OWASP, 2021b).

For a practical illustration, one created a UML flowchart using Visual Paradigm (2019) to demonstrate how threat actors could exploit a security vulnerability in a local cinema chain's system for group bookings. This vulnerability, as highlighted in a scenario by OWASP (2021a), allows users to book up to 15 discounted seats without a deposit, with no validation for the number of seats. Exploiting this, malicious actors can exceed the limit or book multiple cinemas, potentially leading to significant financial losses for the chain.

Naidoo (2024) argues that the point of vulnerability is challenging to work on and ought to be expanded upon, the illustration does demonstrate that there is no security put in place in the vulnerability area to which the flowchart would have illustrated the area of security mitigation, organisation should adopt secure coding practices as Naidoo (2024) pointed out with Comprehensive, Lightweight Application Security Process (CLASP) method of secure coding process (Secure Software Inc., 2024).

Kosar (2024) pointed out that organisations must comprehend the importance of the Software Development Lifecycle (SDLC) to circumvent intricate alterations in the future. A sound security management approach is crucial. The vulnerabilities associated with Insecure Design result from inadequate security measures during the design phase, which can be attributed to the lack of evaluation of business risks (Amazon, n.d.).
